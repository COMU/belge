6'ya 4 tünellerin ayarlarý

2.2.x serisi çekirdeklerin 6'ya 4 tünel desteði olmadýðýný unutmayýn (daha fazla bilgi Çekirdeðin IPv6'ya hazýrlanmasý bölümünde mevcut). Dikkat edilmesi gereken bir baþka konu ise 6'ya 4 adresinin önek uzunluðunun 16 olduðudur. Diðer tüm 6'ya 4 uyumlu konaklar ayný ikinci katmandadýrlar.

6'ya 4 tüneli eklemek

Ýlk olarak yönlendirilebilir IPv4 adresinizi kullanarak 6'ya 4 önekinizi hesaplamalýsýnýz (eðer istemcinizin küresel yönlendirilebilir IPv4 adresi yoksa, özel durumlarda sýnýr að geçitlerinde NAT kullanýlabilir):

IPv4 adresiniz aþaðýdaki gibi ise

   1.2.3.4

oluþtracaðýnýz 6'ya 4 öneki þöyle olacaktýr:

   2002:0102:0304::

Yerel 6'ya 4 að geçitleri ::1 ile sonlanmalýdýr, (eðer daha iyi hissedecekseniz baþka bir þekilde sonlandýrmayý tercih edebilirsiniz) bu yüzden yerel 6'ya 4 adresiniz aþaðýdaki gibi olacaktýr: 

   2002:0102:0304::1

Otomatik üretim aþaðýdakine benzemelidir: 

ipv4="1.2.3.4"; printf "2002:%02x%02x:%02x%02x::1" `echo $ipv4 | tr "." " "`

6to4 tüneli yapýlandýrmak için iki yöntem vardýr.

ip kullanarak ve adanmýþ tünel aygýtý ile
Önerilen yöntem budur (öntanýmlý TTL deðeri 0 olduðundan yenisi atanmalýdýr).

Yeni bir tünel aygýtý oluþturun:

# /sbin/ip tunnel add tun6to4 mode sit ttl öntanýmlý_ttl remote any local yerel_ipv4adres

Arabirimi etkinleþtirin:

# /sbin/ip link set dev tun6to4 up

Arabirime yerel 6'ya 4 adresini atayýn.

Not

Önekin uzunluðunun 16 olmasý önemlidir! 

# /sbin/ip -6 addr add yerel_6ya4_adres /16 dev tun6to4

6ya4 yönlendiricilerini kullanarak küresel IPv6 aðýna (öntanýmlý) yönlendirme ekleyin 

# /sbin/ip -6 route add 2000::/3 via ::192.88.99.1 dev tun6to4 metric 1

ip komutunun bazý sürümlerinin (örneðin SuSE Linux 9.0'da yeralan sürüm) IPv4-uyumlu IPv6 adreslerini að geçidi olarak desteklemediði raporlanmýþtýr. Bu gibi durumlarda ilgili IPv6 adresi kullanýlmalýdýr: 

# /sbin/ip -6 route add 2000::/3 via 2002:c058:6301::1 dev tun6to4 metric 1

ifconfig ve ipkullanarak ve genel tünel aygýtý sit0 ile (önerilmez)
Genel tünel aygýtý sit0 aygýt baþýna filtreleme tanýmlamaya izin vermediðinden bu yöntem önerilmemektedir.

Genel tünel aygýtý sit0'ý etkinleþtirin: 

# /sbin/ifconfig sit0 up

Arabirime yerel 6ya4 adresini ekleyin: 

# /sbin/ifconfig sit0 add yerel_6ya4_adres/16

Tümü 6ya4 röleli IPv4 adreslerini kullanarak küresel IPv6 aðýna yönlendirme ekleyin: 

# /sbin/route -A inet6 add 2000::/3 gw ::192.88.99.1 dev sit0

6ya4 tünelini iptal etmek

ip kullanarak ve adanmýþ tünel aygýtý ile
Adanmýþ tünel aygýtý üzerindeki tüm yönlendirmeleri kaldýrýn: 

# /sbin/ip -6 route flush dev tun6to4

Arabirimi kapatýn: 

# /sbin/ip link set dev tun6to4 down

Mevcut tünel aygýtýný kaldýrýn: 

# /sbin/ip tunnel del tun6to4

ifconfig ve routekullanarak ve genel tünel aygýtý sit0 ile (önerilmez)
6ya4 tünel arabirimi üzerindeki (öntanýmlý) yönlendirmeyi kaldýrýn: 

# /sbin/route -A inet6 del 2000::/3 gw ::192.88.99.1 dev sit0

Arabirimin yerel 6ya4 adresini kaldýrýn: 

# /sbin/ifconfig sit0 del yerel_6ya4_adres /16

Genel tünel aygýtýný kapatýn (dikkatli olun, belki hala kullanýlýyordur): 

# /sbin/ifconfig sit0 down
