X.509 Sertifikalarý

Racoon kimlik denetimi için X.509 sertifikalarýnýn kullanýmýný destekler. Bu sertifikalar bir sertifika otoritesinden (CA) kontrol edilebilirler. Yapýlandýrma PSK yapýlandýrmasýna çok benzer, sadece kimlik kanýtlama bölümü farklýdýr:

	certificate_type x509 "my_certificate.pem" "my_private_key.pem";
	verify_cert on;
	my_identifier asn1dn;
	peers_identifier asn1dn;
	proposal {
		encryption_algorithm 3des;
		hash_algorithm md5;
		authentication_method rsasig;
		dh_group modp1024;
	}
}
					
sainfo address 172.16.1.0/24 any address 172.16.2.0/24 any {
	pfs_group modp768;
	encryption_algorithm 3des;
	authentication_algorithm hmac_md5;
	compression_algorithm deflate;
}

Sertifikalar ve sertifika iptal listeleri openssl ile oluþturulan PEM dosya biçiminde saklanýrlar. Bu sertifikalarýn oluþturulmasýyla ilgili bilgileri X.509 Sertifikalarý Oluþturmak bölümünde bulabilirsiniz. Eðer taraflarýn sertifikalarý bir sertifika otoritesinden kontrol edilecekse (öntanýmlý deðer olan verify_cert on ise) CA'nýn sertifikasý da bu dizinde bulundurulmalýdýr. OpenSSL'in sertifikayý bulabilmesi için ya adý deðiþtirilmeli ya da hash'i alýnmýþ (hashed) dosyaya sembolik bað oluþturulmalýdýr:

Roadwarrior

Roadwarrior bir VPN að geçidine baðlanmak için bilinmeyen dinamik IP adreslerini kullanan istemcilerdir. racoon ile birlikte kullanýmýnda iki problem ortaya çýkar:

  • IP adresi bilinmediðinden racoon yapýlandýrma dosyasýnda veya /etc/psk.txt dosyasýnda belirtilemez. Ýstemcinin kimliðini belirlemenin baþka bir yolu bulunmalýdýr. Önceden paylaþýlmýþ anahtarlar kullanýldýðýnda saldýrgan mod seçilmelidir.En iyi çözüm X.509 sertifikalarýný kullanmak olacaktýr.
  • Hedef IP adresi belirsiz olduðundan racoon için bir güvenlik politikasý yaratýlamaz. racoon güvenlik politikasýný ve güvenlik anlaþmasýný baðlantý baþlatýldýðýnda oluþturmalýdýr.

Bunu yapabilmek için /etc/racoon.conf yapýlandýrma dosyasýnda bazý düzenlemelerin yapýlmasý gereklidir:

path certificate "/etc/certs";
	
remote anonymous {
	exchange_mode main;
	generate_policy on;
	passive on;
	certificate_type x509 "my_certificate.pem" "my_private_key.pem";
	my_identifier asn1dn;
	peers_identifier asn1dn;
	proposal {
	encryption_algorithm 3des;
	hash_algorithm md5;
	authentication_method rsasig;
	dh_group modp1024;
	}
}
	
	
sainfo anonymous {
	pfs_group modp1024;
	encryption_algorithm 3des;
	authentication_algorithm hmac_md5;
	compression_algorithm deflate;
}

generate_policy on seçeneði racoon'a yeni bir baðlantý baþlatýldýðýnda uygun bir politika yaratmasý gerekitiðini bildirir. passive on seçeneði ise racoon'un dýþarýdan yeni bir baðlatý baþlatýlana dek pasif halde kalmasýný söyler. racoon bir baðlantý baþlatmamalýdýr.

Yine de en önemli deðiþiklik remote ve sainfo satýrlarýndaki anonymous tanýmlamasýdýr. Bu sayede racoon her yerden gelen baðlantýlarý kabul edecektir.

Aykýrý NAT

Linux 2.6 çekirdeði tünel modunda aykýrý NAT yapabilme kapasitesine sahiptir. Transfer modu henüz desteklenmemektedir. Bu ipsec-tools'un 0.3.3 sürümünden itibaren Racoon ile yapýlabilir.

Racooon'un aykýrý NAT için yapýlandýrýlabilmesi için yapýlandýrma dosyasýna bir çok seçenek eklenmiþtir. Bunlar: natt_keepalive, isakmp_natt, nat_traversal dir.

En önemli seçenek nat_traversal'dir. Bu seçeneðe deðer olarak on, off veya force deðerleri atanabilir. On atandýðýnda, yol üzerinde bir NAT aygýtý algýlanýr algýlanmaz aykýrý NAT kullanýlmaya baþlanýr. Off bu özelliðin kullanýlmasýný iptal eder. force kullanýldýðýnda ise bir NAT aygýtý bulunsun veya bulunmasýn aykýrý NAT kullanýlacaktýr.

NAT aygýtlarýnýn çoðu bir trafik olmadýðýnda hýzlý bir þekilde kendi iç tablolarýndaki girdileri unuttuklarýndan, racoon aðdan keepalive paketleri göndermeyi önerir. Bu paketler ön tanýmlý olarak her 20 saniyede bir gönderilirler. Bu deðeri natt_keepalive deðiþkenini kullanarak deðiþtirebilirsiniz. Eðer deðer olarak 0 verirseniz bu özelliði kapatmýþ olursunuz.

Aykýrý NAT kullanmak isterseniz racoon'un yapýlandýrma dosyasýnýn listen bölümünde kullanýlacak IP adresini ve portu belirtmeniz gerekir. Bu isakmp_natt kullanýlarak yapýlýr.

Örnek olmasý için aþaðýda 192.168.2.100 adresli tarafýn 192.168.1.1 adresine sahip NAT aygýtý arkasýnda olduðu bir durumun yapýlandýrma dosyasý verilmektedir:s

	natt_keepalive 10sec;
	}

	isakmp 192.168.1.100 [500];
	isakmp_natt 192.168.1.100 [4500];
}
			
remote 192.168.1.1 {
	exchange_mode main;
	nat_traversal on;
	proposal {
		encryption_algorithm 3des;
		hash_algorithm md5;
		authentication_method pre_shared_key;
		dh_group modp1024;
	}
}
			
sainfo address 172.16.1.0/24 any address 172.16.2.0/24 any {
	pfs_group modp768;
	encryption_algorithm 3des;
	authentication_algorithm hmac_md5;
