Ýmleme Sýnama ACL'si

Bu þemanýn son parçasý, Exim'e imli olarak geçerli alýcý adreslere gelen teslimatlarýn daima kabul edileceðini ve boþ gönderici adresli diðer teslimatlarýn ise eðer alýcý bu þemayý seçmiþse reddedileceðini belirtmektir. Böyle durumlarda grilisteleme yapýlmamalýdýr.

Aþaðýdaki satýrlarý acl_rcpt_to ACL'sinde olasý bir SPF, grilisteleme ve/veya sonuncu accept deyiminin öncesine yerleþtirin: 

  # Kendine özgü imlemesini içeriyorsa, alýcý adresini kabul ediyoruz.
  # Bu, teslimatýn, daha önce bizden gönderilmiþ bir postanýn teslimat
  # durum bildirimi olduðunu gösterir.
  #
  accept
    domains     = +local_domains
    condition   = ${if and {{match{${lc:$local_part}}{^(.*)=(.*)}}\
                            {eq{${hash_8:${hmac{md5}{SECRET}{$1}}}}{$2}}}\
                           {true}{false}}

  # Aksi takdirde, posta boþ gönderici adresli ise ama alýcý, imlemeli zarf
  # gönderici adresi þemasýný seçenlerden biri ise postayý reddediyoruz.
  #
  deny
    message     = This address does not match a valid, signed \
                  return path from here.\n\
                  You are responding to a forged sender address.
    log_message = bogus bounce.
    senders     = : postmaster@*
    domains     = +local_domains
    set acl_m9  = /home/${extract{1}{=}{${lc:$local_part}}}/.return-path-sign
    condition   = ${if exists {$acl_m9}{true}}

Postayý gönderirken iletinin baþlýðýndaki adreslere (örneðin, gönderdiðiniz postanýn From: alanýndaki adrese) varlýk doðrulamasý yapan konaklarýn varlýðýný bu noktada dikkate almak gerekir. Buradaki deny deyimi normal olarak böyle bir doðrulatma çabasýna olumsuz yanýt verecektir.

Bunun olmamasý için, deny deyimini warn deyimi haline getirmek, red iletisini $acl_m0 deðiþkeninde saklayýp asýl reddi DATA komutundan sonra yapmak isteyebilirsiniz: 

  # Aksi takdirde, posta boþ gönderici adresli ise ama alýcý, imlemeli zarf
  # gönderici adresi þemasýný seçenlerden biri ise, red iletisini $acl_m0
  # ve günlük iletisini $acl_m1 deðiþkenine kaydedip, bunlarý daha sonra
  # postayý reddederken kullanacaðýz. Red sýrasýnda göndericinin oyalanýp
  # oyalanmayacaðýna bunlarýn varlýðýna bakarak karar vereceðiz.
  #
  warn
    senders     = : postmaster@*
    domains     = +local_domains
    set acl_m9  = /home/${extract{1}{=}{${lc:$local_part}}}/.return-path-sign
    condition   = ${if exists {$acl_m9}{true}}
    set acl_m0  = The recipient address <$local_part@$domain> does not \
                  match a valid, signed return path from here.\n\
                  You are responding to a forged sender address.
    set acl_m1  = bogus bounce for <$local_part@$domain>.

Ayrýca, alýcý giden postasýnda imlemeli zarf gönderici adresi kullanmayý seçmiþ bile olsa, bazý konaklardan gelen postalarýn zarf gönderici adresi boþ olsa bile sýnama dýþý tutulmasýný isteyebilir. Bilhassa eposta listelerinin sunucularý buna en iyi örnektir, bu konuda daha ayrýntýlý bilgi için Zarf Gönderici Ýmleri bölümüne bakýnýz.
